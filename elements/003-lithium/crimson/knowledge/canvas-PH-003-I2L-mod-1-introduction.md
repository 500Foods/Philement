Welcome to the ***Introduction to Lithium*** course on [500courses.com](https://www.500courses.com).

Lithium is the project codename for a client web application - the frontend - that is used to interact with a hydrogen server - the backend. Some projects use the name Lithium directly, particularly in the 3D printing context. Other projects use the same combination of components, but with branding specific to that project. [GAIUS](https://www.gaiusmodel.com) is one such example of this, but there are others discussed in detail in the next section - Lithium Clients.

So what is Lithium? 

In broad terms, it is a traditional web app. People login, move between different sections of the app, enter data, run reports,  use dashboards, get notifications, all the usual sorts of things. The actual components within Lithium that *you* use might be very different than what a coworker or colleague might use given the different roles you might play in your organization. Some people might be working more with dashboards, monitoring an organization\'s KPIs at a very high level. Others might be doing basic data entry or traditional accounting-type  tasks. There may be little overlap between them. It all depends on what access has been granted and what Lithium features an organization has opted to use.

This course, then, has been designed as a tool to help people get up to speed on how to get the most from Lithium. But with an eye towards providing a path through the material for the particular Lithium implementation that your organization is using. And that organization might be very small - a single standalone 3D printer can run both Lithium and Hydrogen all on its own in a self-contained environment, designed to be accessible by a single user. Or it can be used by multinational organizations with thousands of daily users. We\'ve got you covered!

For someone new to Lithium, minimal onboarding would involve covering the Common Lithium Elements material and then the specific Managers that you\'ll be using day-to-day. But unlike other courses, there\'s no prescribed path through the material - all modules are available right now. You can start at the beginning and work your way through to the end in the published order, or jump ahead to specific sections. Each module has generally been written to be consumed on its own, with plenty of links to other sections where needed.

So dive in and by all means ask questions or leave feedback if you find something isn\'t covered in sufficient depth for your situation.
